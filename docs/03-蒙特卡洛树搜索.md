# 蒙特卡洛树搜索：让电脑学会"往后想"

> 这是第三篇教程。上一篇我们给电脑装上了"棋感"（神经网络）。但光有直觉还不够——真正的高手会往后推演好几步。蒙特卡洛树搜索（MCTS）就是让电脑学会"往后想"的方法。

## 人类高手怎么下棋？

你下棋的时候，遇到关键局面会在脑子里推演：

"如果我下这里，对手可能下那里，然后我再下这里……"

你会在脑子里画出一棵"决策树"——每一步都有好几个选择，每个选择又引出对手的好几个应对。你沿着几条看起来最有希望的路线往下想，最后选出最好的那步。

MCTS 做的事情一模一样，只不过它不是"想"，而是真的在电脑里模拟成千上万次。

## 为什么叫"蒙特卡洛"？

蒙特卡洛是摩纳哥的一个城市，以赌场闻名。叫这个名字是因为 MCTS 的核心思想和赌博很像——**靠大量随机试验来估算概率**。

举个例子。你想知道一个骰子是不是公平的，最直接的办法不是去分析它的物理结构，而是扔它一万次，看看每个面出现的次数。这就是"蒙特卡洛方法"——用随机试验来逼近答案。

MCTS 也是这个思路：与其精确计算所有可能性（计算量太大），不如随机模拟很多局，用统计结果来估算每步棋的好坏。

## MCTS 的四个步骤

MCTS 每一轮搜索都重复四个步骤，就像一个人反复做"选路线 → 走到头 → 看结果 → 记笔记"的循环。

### 第一步：选择（Selection）

从当前棋盘开始，沿着之前探索过的路线往下走，一直走到一个还没探索过的局面。

这就像你站在一个迷宫入口，之前已经走过几条路了。你会优先走那些"看起来有希望但还没走到底"的路线。

MCTS 用一个叫 UCB1 的公式来决定走哪条路。这个公式平衡两件事：

- **利用**：走之前赢过很多次的路线（"这条路好像不错"）
- **探索**：走之前没怎么试过的路线（"万一这条路更好呢？"）

就像你选餐厅：是去吃过觉得不错的那家，还是试试新开的？UCB1 帮你在两者之间找到平衡。

### 第二步：扩展（Expansion）

走到一个从没见过的局面时，把它加入搜索树，然后随机选一步棋走下去。

这就像你在迷宫里走到了一个新岔路口，先把它标记在地图上，然后随便选一条路继续走。

### 第三步：模拟（Simulation）

从新加入的局面开始，双方都随机下棋，一直下到游戏结束。

这就像你在新岔路口随便选了一条路之后，一路瞎走到迷宫出口，看看走出去没有。

注意：模拟阶段是**完全随机**的，不用神经网络，也不用任何策略。就是双方乱下，看谁赢。一局模拟可能只需要几毫秒，因为井字棋最多 9 步就结束了。

### 第四步：回溯（Backpropagation）

模拟结束后，把结果（谁赢了）沿着刚才走过的路线一路传回去，更新每个节点的统计信息。

这就像你从迷宫出口原路返回，在每个岔路口做个记号："从这个方向走，最终走出去了"或者"从这个方向走，最终走进了死胡同"。

下次再经过这些岔路口时，你就能根据这些记号做出更好的选择。

## 一个具体的例子

假设当前棋盘是这样的，轮到 X 下：

```
 X | O |
-----------
   | X |
-----------
   |   |
```

X 有三个空位可以下：位置 2、3、5、6、7、8（共 6 个选择）。MCTS 会这样做：

**第 1 轮搜索**：选位置 2 → 随机模拟 → X 赢了 → 位置 2 的记录变成"1 胜 0 负"

**第 2 轮搜索**：选位置 3 → 随机模拟 → 平局 → 位置 3 的记录变成"0 胜 1 平"

**第 3 轮搜索**：选位置 5 → 随机模拟 → X 赢了 → 位置 5 的记录变成"1 胜 0 负"

**第 4 轮搜索**：UCB1 发现位置 2 和 5 赢过，但位置 6、7、8 还没试过 → 选位置 6 → 随机模拟 → O 赢了 → 位置 6 的记录变成"0 胜 1 负"

……

**第 50 轮搜索结束**，统计结果可能是：

```
位置 2:  18 次访问，胜率 72%  ← 看起来不错
位置 3:   8 次访问，胜率 50%
位置 5:  15 次访问，胜率 67%
位置 6:   4 次访问，胜率 25%  ← 不太好
位置 7:   3 次访问，胜率 33%
位置 8:   2 次访问，胜率 50%
```

MCTS 会选访问次数最多的位置 2——因为它被探索得最充分，统计结果最可靠。

## 神经网络怎么帮 MCTS？

纯 MCTS 的模拟阶段是完全随机的，就像两个不会下棋的人在乱下。这样模拟出来的结果不太靠谱。

AlphaGo 的改进是：用神经网络来指导搜索。

**选择阶段**：不再用 UCB1 的简单公式，而是用神经网络的策略头来优先探索"棋感"好的位置。就像让一个有经验的棋手来选"哪些路线值得深入想"。

对应的代码实现：

```python
def ucb_score(self, c_puct=1.4):
    if self.visits == 0:
        exploration = float('inf')  # 没试过的优先试
    else:
        parent_visits = self.parent.visits if self.parent else 1
        # 利用项：平均胜率
        # 探索项：先验概率 × sqrt(父节点访问次数) / (1 + 自身访问次数)
        exploration = c_puct * self.prior * math.sqrt(parent_visits) / (1 + self.visits)
    return self.value + exploration
```

注意这里的 `prior`（先验概率）就是神经网络策略头输出的。标准 UCB1 没有这一项——它对所有未探索的路线一视同仁。而 AlphaGo 的版本让神经网络告诉我们"哪些路线更值得优先看"，搜索效率大大提高。

**模拟阶段**：不再完全随机下棋，而是用神经网络的价值头直接评估当前局面。就像不用真的下到最后一子，一个高手看一眼棋盘就能判断谁占优。

```python
# 用神经网络评估叶子节点
state = torch.FloatTensor(sim_board.to_tensor()).unsqueeze(0)
with torch.no_grad():
    policy_logits, value = net(state)

leaf_value = value.item()  # 价值头：这个局面对当前玩家有多好
```

纯 MCTS 必须随机下到游戏结束才知道谁赢，而 AlphaGo 的 MCTS 可以在任何局面直接"看一眼"就给出评分。这就像让一个职业棋手来评估局面，比两个新手乱下得出的结果靠谱得多。

**回溯阶段**：把价值头的评分沿路径传回去，每经过一层就翻转符号（因为对手的好处就是我的坏处）。

```python
current = node
v = leaf_value
while current is not None:
    current.visits += 1
    current.value_sum += v
    v = -v  # 翻转视角：我的好事是对手的坏事
    current = current.parent
```

## 完整的 MCTS 搜索循环

把上面三步合在一起，就是完整的 MCTS 搜索函数：

```python
def mcts_search(board, net, num_simulations=100):
    root = MCTSNode()

    for _ in range(num_simulations):
        node = root
        sim_board = board.copy()

        # 1. 选择：沿 UCB 最高的路径走到叶子
        while node.children and sim_board.check_winner() is None:
            best_move = max(node.children, key=lambda m: node.children[m].ucb_score())
            node = node.children[best_move]
            sim_board.play(best_move)

        # 2. 扩展：用神经网络评估叶子节点
        winner = sim_board.check_winner()
        if winner is not None:
            leaf_value = 0 if winner == 0 else (1 if winner == board.current_player else -1)
        else:
            state = torch.FloatTensor(sim_board.to_tensor()).unsqueeze(0)
            with torch.no_grad():
                policy_logits, value = net(state)
            leaf_value = value.item()
            # 创建子节点，用策略头输出作为先验概率
            policy = torch.softmax(policy_logits, dim=1).squeeze().numpy()
            for move in sim_board.legal_moves():
                child = MCTSNode(parent=node, move=move)
                child.prior = policy[move]
                node.children[move] = child

        # 3. 回溯：把价值传回根节点，每层翻转符号
        current = node
        v = leaf_value
        while current is not None:
            current.visits += 1
            current.value_sum += v
            v = -v
            current = current.parent

    # 返回每个走法的访问次数分布
    return {move: root.children[move].visits if move in root.children else 0
            for move in board.legal_moves()}
```

搜索结束后，MCTS 返回每个合法走法的**访问次数分布**。访问次数越多的走法，说明被探索得越充分、统计结果越可靠。这个分布就是 MCTS 给出的"改进后的策略"——比神经网络单独给出的策略更准确，因为它经过了大量推演验证。

## 小结

MCTS 的核心思想很简单：**与其精确计算所有可能性，不如随机模拟很多局，用统计结果来估算每步棋的好坏。**

| 步骤 | 做什么 | 类比 |
|------|--------|------|
| 选择 | 沿 UCB 最高的路径走到叶子 | 选最有希望的路线深入想 |
| 扩展 | 用神经网络评估新局面 | 让高手看一眼判断好坏 |
| 回溯 | 把评估值传回路径上所有节点 | 在岔路口做记号 |

加上神经网络的指导后，MCTS 不再是盲目的随机模拟，而是有方向的深度推演。神经网络提供"直觉"，MCTS 提供"推演"，两者结合就是 AlphaGo 的核心能力。
