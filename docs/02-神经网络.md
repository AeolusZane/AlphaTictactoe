# 神经网络：给电脑装上"棋感"

> 这是第二篇教程。上一篇我们教会了电脑认识棋盘和规则。但光认识规则还不够——电脑需要学会"看"一眼棋盘，就能判断该下哪里、谁占优势。这就是神经网络的工作。

## 人类下棋靠什么？

你下井字棋的时候，不会把后面所有可能的走法都想一遍。你看到棋盘，脑子里会冒出两个直觉：

第一个直觉是"棋感"——扫一眼就知道哪几个位置看起来不错，值得考虑。比如对手快连成三个了，你本能地想去堵。

第二个直觉是"大局观"——看一眼局面，你能感觉到"这盘我占优"或者"要输了"。你不需要算到最后一步，凭经验就能判断。

AlphaGo 的神经网络就是在模仿这两个直觉。它有两个"头"：一个负责"棋感"（策略头），一个负责"大局观"（价值头）。

## 策略头：该下哪里？

策略头的输入是当前棋盘，输出是 9 个数字，代表每个位置"值得下"的程度。

举个例子。假设当前棋盘是这样的：

```
 X | O |
-----------
   | X |
-----------
   |   | O
```

策略头可能会输出：

```
位置 0:  已经下了 X
位置 1:  已经下了 O
位置 2:  0.35  ← 比较高，值得考虑
位置 3:  0.05
位置 4:  已经下了 X
位置 5:  0.15
位置 6:  0.30  ← 也比较高
位置 7:  0.10
位置 8:  已经下了 O
```

数字越大，表示这个位置越值得下。位置 2 和位置 6 得分最高——如果你仔细看棋盘，会发现 X 在位置 2 或 6 下子就能形成对角线威胁。这就是"棋感"。

注意：已经下过的位置不用管，策略头只关心空位。

## 价值头：谁占优势？

价值头的输入也是当前棋盘，但输出只有一个数字，范围在 -1 到 1 之间：

| 输出值 | 含义 |
|--------|------|
| 接近 +1 | 当前玩家大概率会赢 |
| 接近 0 | 势均力敌，可能平局 |
| 接近 -1 | 当前玩家大概率会输 |

还是刚才那个棋盘：

```
 X | O |
-----------
   | X |
-----------
   |   | O
```

如果当前轮到 X 下，价值头可能输出 0.6——意思是"X 占优势，但还没赢"。这很合理，因为 X 已经占了中间和两个角，局面不错。

## 神经网络长什么样？

别被"神经网络"这个词吓到。我们的网络结构非常简单，就像一条流水线：

```
棋盘状态（9个数字）
       ↓
   第一层：把 9 个数字变成 64 个数字
       ↓
   第二层：64 个数字互相"交流"
       ↓
   第三层：64 个数字继续"交流"
       ↓
   分叉成两个头：
   ├── 策略头：64 → 9（每个位置的概率）
   └── 价值头：64 → 1（谁占优势）
```

每一层做的事情，其实就是把数字乘上一堆"权重"再加起来。这些权重就是神经网络的"记忆"——训练的过程，就是不断调整这些权重，让输出越来越准。

打个比方：你学骑自行车的时候，大脑也在不断调整"权重"——身体往左倾多少、手把往右拐多少。一开始总是歪歪扭扭，骑多了就稳了。神经网络也一样，一开始乱猜，训练多了就准了。

## 输入格式：为什么是 2×9？

棋盘有 9 个格子，但我们给神经网络的输入不是 9 个数字，而是 18 个。为什么？

因为我们把棋盘拆成了两层：

```
第一层（我的棋子在哪里）：
 1  0  0
 0  1  0
 0  0  0

第二层（对手的棋子在哪里）：
 0  1  0
 0  0  0
 0  0  1
```

第一层标记"我"的棋子位置（有就是 1，没有就是 0），第二层标记"对手"的棋子位置。这样神经网络能更清楚地分辨"哪个是我的、哪个是对方的"。

这就像看地图的时候，用红色标自己的阵地、蓝色标敌人的阵地，比只用一种颜色清楚得多。

## 训练前 vs 训练后

神经网络刚创建的时候，所有权重都是随机的。这时候它的表现就像闭着眼睛乱猜：

```
训练前（随机权重）：
  策略头输出：每个位置都差不多，0.11 左右（完全不知道该下哪）
  价值头输出：接近 0（不知道谁占优）
```

训练 200 局之后：

```
训练后：
  策略头输出：能准确指出最佳位置，概率分布有明显的高低
  价值头输出：能判断局面优劣，接近真实胜负
```

这个"从乱猜到准确"的过程，就是训练。具体怎么训练，我们会在第四篇教程里详细讲。

## 代码在哪里？

对应代码中的 `PolicyValueNet` 类：

```python
class PolicyValueNet(nn.Module):
    def __init__(self):
        # 定义网络结构：3层 + 2个头
        self.fc1 = nn.Linear(18, 64)    # 输入层
        self.fc2 = nn.Linear(64, 64)    # 隐藏层
        self.fc3 = nn.Linear(64, 64)    # 隐藏层
        self.policy_head = nn.Linear(64, 9)   # 策略头
        self.value_head = nn.Linear(64, 1)    # 价值头

    def forward(self, x):
        # 数据流过网络
        x = F.relu(self.fc1(x))
        x = F.relu(self.fc2(x))
        x = F.relu(self.fc3(x))
        policy = self.policy_head(x)    # 输出"该下哪"
        value = torch.tanh(self.value_head(x))  # 输出"谁占优"
        return policy, value
```

`F.relu` 是一个简单的数学函数，作用是让网络能学到更复杂的模式。你可以把它想象成流水线上的一个"加工站"——每经过一个站，数字就被加工一次，变得越来越有用。

## 小结

| 概念 | 类比 | 作用 |
|------|------|------|
| 策略头 | 棋感 | 告诉你"哪些位置值得下" |
| 价值头 | 大局观 | 告诉你"当前谁占优势" |
| 权重 | 骑自行车的经验 | 通过训练不断调整，越来越准 |
| 2×9 输入 | 红蓝两色地图 | 让网络分清"我的"和"对手的" |

神经网络给了我们两个能力：看棋盘知道该下哪（策略），看棋盘知道谁占优（价值）。但光靠它还不够——它只是"直觉"，有时候直觉会出错。下一篇我们会讲蒙特卡洛树搜索（MCTS），它就像一个"理性思考"的过程，能验证和修正直觉，做出更可靠的决策。
