# 神经网络：给电脑装上"棋感"

> 这是第二篇教程。上一篇我们教会了电脑认识棋盘和规则。但光认识规则还不够——电脑需要学会"看"一眼棋盘，就能判断该下哪里、谁占优势。这就是神经网络的工作。

## 人类下棋靠什么？

你下井字棋的时候，不会把后面所有可能的走法都想一遍。你看到棋盘，脑子里会冒出两个直觉：

第一个直觉是"棋感"——扫一眼就知道哪几个位置看起来不错，值得考虑。比如对手快连成三个了，你本能地想去堵。

第二个直觉是"大局观"——看一眼局面，你能感觉到"这盘我占优"或者"要输了"。你不需要算到最后一步，凭经验就能判断。

AlphaGo 的神经网络就是在模仿这两个直觉。它有两个"头"：一个负责"棋感"（策略头），一个负责"大局观"（价值头）。

## 策略头：该下哪里？

策略头的输入是当前棋盘，输出是 9 个数字，代表每个位置"值得下"的程度。

举个例子。假设当前棋盘是这样的：

```
 X | O |
-----------
   | X |
-----------
   |   | O
```

策略头可能会输出：

```
位置 0:  已经下了 X
位置 1:  已经下了 O
位置 2:  0.35  ← 比较高，值得考虑
位置 3:  0.05
位置 4:  已经下了 X
位置 5:  0.15
位置 6:  0.30  ← 也比较高
位置 7:  0.10
位置 8:  已经下了 O
```

数字越大，表示这个位置越值得下。位置 2 和位置 6 得分最高——如果你仔细看棋盘，会发现 X 在位置 2 或 6 下子就能形成对角线威胁。这就是"棋感"。

注意：已经下过的位置不用管，策略头只关心空位。

## 价值头：谁占优势？

价值头的输入也是当前棋盘，但输出只有一个数字，范围在 -1 到 1 之间：

| 输出值 | 含义 |
|--------|------|
| 接近 +1 | 当前玩家大概率会赢 |
| 接近 0 | 势均力敌，可能平局 |
| 接近 -1 | 当前玩家大概率会输 |

还是刚才那个棋盘：

```
 X | O |
-----------
   | X |
-----------
   |   | O
```

如果当前轮到 X 下，价值头可能输出 0.6——意思是"X 占优势，但还没赢"。这很合理，因为 X 已经占了中间和两个角，局面不错。

## 神经网络长什么样？

别被"神经网络"这个词吓到。我们的网络结构非常简单，就像一条流水线：

```
棋盘状态（18个数字 = 2层 × 9格）
       ↓
   第一层：把 18 个数字变成 64 个数字
       ↓
   第二层：64 个数字互相"交流"
       ↓
   第三层：64 个数字继续"交流"
       ↓
   分叉成两个头：
   ├── 策略头：64 → 9（每个位置的概率）
   └── 价值头：64 → 1（谁占优势）
```

> **64 是怎么来的？**
>
> 不是算出来的，是**人为选定**的。网络中各维度的来源不同：
>
> | 维度 | 来源 | 是否固定 |
> |------|------|---------|
> | **18**（输入） | 2 层 × 9 格（我的棋子 + 对手的棋子） | 由棋盘大小决定 |
> | **9**（策略头输出） | 棋盘 9 个位置 | 由棋盘大小决定 |
> | **1**（价值头输出） | 一个胜负判断值 | 固定 |
> | **64**（隐藏层） | 人为设定 | 可以改成任意值 |
>
> 隐藏层维度的选择通常遵循两个原则：一是**比输入大**，给网络足够的"脑容量"来学习复杂模式；二是**惯用值**，64、128、256 是常见选择（2 的幂次便于计算，且经过大量实践验证效果不错）。
>
> 你可以把 64 换成 32 或 128 试试——32 可能学得太慢，128 可能学得更快但容易"死记硬背"（过拟合）。64 是一个在井字棋这种简单问题上够用的平衡点。

## 隐藏层：网络是怎么"思考"的？

上面的结构图里，第一层、第二层、第三层都叫**隐藏层**。"隐藏"这个名字听起来很神秘，其实意思很简单——它们是夹在输入和输出之间的中间层，你从外面看不到它们在干什么，所以叫"隐藏"。

那这些中间层到底在干什么？

### 一个比喻：智囊团

想象你是一个将军，面前有一张战场地图（输入）。你需要做两个决策：派兵去哪里（策略），以及判断谁占优势（价值）。

你不会自己盯着地图硬想。你有一群参谋（隐藏层），每个人负责分析不同的方面：

```
战场地图（输入：18个数字）
       ↓
   参谋 A：分析"我方兵力分布是否形成包围"
   参谋 B：分析"敌方有没有薄弱环节"
   参谋 C：分析"关键地形谁占了"
   参谋 D：分析"补给线是否安全"
   ...（一共 64 个参谋）
       ↓
   参谋们互相讨论、交换意见（第二层、第三层）
       ↓
   汇总报告：
   ├── 策略建议：建议进攻方向（9个位置的概率）
   └── 局势判断：我方占优程度（-1 到 1）
```

每个"参谋"就是隐藏层中的一个**神经元**。64 个神经元 = 64 个参谋，每个参谋从不同角度分析棋盘。

### 神经元在做什么？

每个神经元做的事情非常简单：**把输入的数字乘上各自的权重，加起来，得到一个结果**。

```
神经元的工作（以第一层的某个神经元为例）：

输入：18 个数字（棋盘状态）
       ↓
   数字1 × 权重1
 + 数字2 × 权重2
 + 数字3 × 权重3
 + ...
 + 数字18 × 权重18
       ↓
   加起来 = 这个神经元的"判断"
```

权重决定了这个神经元"关注什么"。比如：

- 某个神经元给"中心位置有我的棋子"很大的权重 → 它专门关注"是否占了中心"
- 另一个神经元给"对手快连成三个"很大的权重 → 它专门关注"是否需要防守"
- 还有一个神经元给"两个角都是我的"很大的权重 → 它专门关注"有没有形成包围"

**关键点**：这些"关注点"不是我们告诉网络的，是网络在训练过程中自己发现的。就像你不需要告诉每个参谋"你负责看地形"——他们会在实践中自然分工。

### 为什么要多层？

你可能想：一层参谋不就够了，为什么要三层？

因为有些判断需要"层层递进"：

```
第一层参谋：发现"位置 2 有我的棋子"和"位置 4 有我的棋子"
                ↓
第二层参谋：综合第一层的发现 → "哦，2 和 4 形成了一条斜线的威胁"
                ↓
第三层参谋：综合第二层的发现 → "这条斜线再加一个子就能赢了，优先级很高！"
```

第一层看到的是**局部特征**（某个位置有没有棋子）。
第二层看到的是**组合特征**（几个位置之间的关系）。
第三层看到的是**全局特征**（整盘棋的战略态势）。

层数越多，网络能理解的"逻辑"就越复杂。但井字棋很简单，三层就够了。围棋那种复杂游戏可能需要几十层甚至上百层（所以叫"深度学习"——层数多就是"深"）。

### ReLU：给神经元加个"开关"

代码里有个 `F.relu`，它的全名是 Rectified Linear Unit（修正线性单元），名字很唬人，但做的事情极其简单：

```
如果结果 > 0 → 原样输出
如果结果 ≤ 0 → 输出 0
```

就这么简单。它的作用是让神经元有一个"开关"——只有当某个特征足够明显时才"激活"，否则就沉默。

打个比方：你走在路上，闻到一点香味（神经元算出来是 0.3），你不会停下脚步。但如果香味很浓（算出来是 5.0），你就会转头去找是哪家店。ReLU 就是这个"够浓才反应"的机制。

如果没有 ReLU，不管多微弱的信号都会往下传，网络就会被噪音淹没，学不到有用的东西。

---

每一层做的事情，其实就是把数字乘上一堆"权重"再加起来。这些权重就是神经网络的"记忆"——训练的过程，就是不断调整这些权重，让输出越来越准。

打个比方：你学骑自行车的时候，大脑也在不断调整"权重"——身体往左倾多少、手把往右拐多少。一开始总是歪歪扭扭，骑多了就稳了。神经网络也一样，一开始乱猜，训练多了就准了。

## 输入格式：为什么是 2×9？

棋盘有 9 个格子，但我们给神经网络的输入不是 9 个数字，而是 18 个。为什么？

因为我们把棋盘拆成了两层：

```
第一层（我的棋子在哪里）：
 1  0  0
 0  1  0
 0  0  0

第二层（对手的棋子在哪里）：
 0  1  0
 0  0  0
 0  0  1
```

第一层标记"我"的棋子位置（有就是 1，没有就是 0），第二层标记"对手"的棋子位置。这样神经网络能更清楚地分辨"哪个是我的、哪个是对方的"。

这就像看地图的时候，用红色标自己的阵地、蓝色标敌人的阵地，比只用一种颜色清楚得多。

## 训练前 vs 训练后

神经网络刚创建的时候，所有权重都是随机的。这时候它的表现就像闭着眼睛乱猜：

```
训练前（随机权重）：
  策略头输出：每个位置都差不多，0.11 左右（完全不知道该下哪）
  价值头输出：接近 0（不知道谁占优）
```

训练 200 局之后：

```
训练后：
  策略头输出：能准确指出最佳位置，概率分布有明显的高低
  价值头输出：能判断局面优劣，接近真实胜负
```

这个"从乱猜到准确"的过程，就是训练。具体怎么训练，我们会在第四篇教程里详细讲。

## 代码在哪里？

对应代码中的 `PolicyValueNet` 类：

```python
class PolicyValueNet(nn.Module):
    def __init__(self):
        # 定义网络结构：3层 + 2个头
        self.fc1 = nn.Linear(18, 64)    # 输入层
        self.fc2 = nn.Linear(64, 64)    # 隐藏层
        self.fc3 = nn.Linear(64, 64)    # 隐藏层
        self.policy_head = nn.Linear(64, 9)   # 策略头
        self.value_head = nn.Linear(64, 1)    # 价值头

    def forward(self, x):
        # 数据流过网络
        x = F.relu(self.fc1(x))
        x = F.relu(self.fc2(x))
        x = F.relu(self.fc3(x))
        policy = self.policy_head(x)    # 输出"该下哪"
        value = torch.tanh(self.value_head(x))  # 输出"谁占优"
        return policy, value
```

`F.relu` 是一个简单的数学函数，作用是让网络能学到更复杂的模式。你可以把它想象成流水线上的一个"加工站"——每经过一个站，数字就被加工一次，变得越来越有用。

## 小结

| 概念 | 类比 | 作用 |
|------|------|------|
| 隐藏层 | 智囊团 | 从不同角度分析棋盘，层层递进 |
| 神经元 | 参谋 | 每个关注一个特征（占中心、防威胁...） |
| 策略头 | 棋感 | 告诉你"哪些位置值得下" |
| 价值头 | 大局观 | 告诉你"当前谁占优势" |
| 权重 | 骑自行车的经验 | 通过训练不断调整，越来越准 |
| ReLU | 够浓才反应的嗅觉 | 过滤噪音，只让有用的信号通过 |
| 2×9 输入 | 红蓝两色地图 | 让网络分清"我的"和"对手的" |

神经网络给了我们两个能力：看棋盘知道该下哪（策略），看棋盘知道谁占优（价值）。但光靠它还不够——它只是"直觉"，有时候直觉会出错。下一篇我们会讲蒙特卡洛树搜索（MCTS），它就像一个"理性思考"的过程，能验证和修正直觉，做出更可靠的决策。

## 扩展阅读：隐藏层维度的选择

前面提到隐藏层用了 64 这个"魔法数字"。这里展开讲讲它背后的考量——虽然对井字棋这种简单问题，64 还是 128 差别不大，但理解这个选择过程对以后做更复杂的项目很有帮助。

### 为什么是 2 的幂次？

64 = 2^6。深度学习里常见的隐藏层维度几乎都是 2 的幂次：32、64、128、256、512、1024。这不是巧合，有几个实际原因：

**1. 计算便利**

神经网络的核心运算是矩阵乘法。当维度是 2 的幂次时，内存分配更整齐，计算更高效。举个例子：

```python
# 维度是 64 时
x = torch.randn(batch_size, 64)  # 内存对齐自然
W = torch.randn(64, 64)          # 矩阵运算整齐

# 维度是 63 时
x = torch.randn(batch_size, 63)  # 内存可能有碎片
W = torch.randn(63, 63)          # 计算效率略低
```

差别很小，但当成千上万次矩阵运算累积起来，2 的幂次能省下不少时间。

**2. 容量足够**

隐藏层的作用是"思考空间"——它需要比输入和输出都大，才能学到复杂的模式。

我们的网络：
- 输入：18（2×9 棋盘状态）
- 隐藏：64（思考空间）
- 输出：9（策略）+ 1（价值）

64 比 18 和 9 都大，网络有足够的"脑容量"来记住井字棋的各种局面和应对策略。

**3. 任务复杂度匹配**

不同复杂度的任务需要不同大小的"脑容量"：

| 任务 | 状态空间 | 典型隐藏层维度 |
|------|---------|---------------|
| 井字棋 | 最多 9! ≈ 36 万种局面 | 64 / 128 |
| 五子棋 | 15×15 棋盘，局面极多 | 256 / 512 |
| 围棋 | 19×19 棋盘，局面天文数字 | 512 / 1024 / 2048 |
| 图像识别 | 百万像素，千变万化 | 1024 / 2048 / 4096 |

井字棋很简单，64 个神经元足够记住所有关键模式（比如"对手快赢了要堵"、"形成双威胁必胜"）。如果用 1024，就像用大炮打蚊子——能打赢，但浪费资源。

**4. 行业惯例**

从 2012 年 AlexNet 到 2015 年 ResNet，深度学习社区逐渐形成了一套"标准配置"：

- **AlexNet (2012)**：4096（当时 GPU 内存小，只能做到这么大）
- **VGG (2014)**：4096（延续了大维度的思路）
- **ResNet (2015)**：64 / 128 / 256（发现小维度 + 深网络效果更好）

ResNet 之后，64/128/256 成了主流选择。大家用习惯了，新论文也沿用这套配置，形成了良性循环——别人用 64 效果不错，我也用 64，省得自己调参。

### 怎么调参？

如果你不确定该用多大的隐藏层，标准做法是**实验对比**：

```python
# 实验 1：隐藏层 = 32
model_small = PolicyValueNet(hidden_dim=32)
train(model_small)  # 训练 200 局，记录胜率

# 实验 2：隐藏层 = 64
model_medium = PolicyValueNet(hidden_dim=64)
train(model_medium)  # 训练 200 局，记录胜率

# 实验 3：隐藏层 = 128
model_large = PolicyValueNet(hidden_dim=128)
train(model_large)  # 训练 200 局，记录胜率

# 对比结果
# 32:  训练慢，胜率 45%（学得太慢，没记住关键模式）
# 64:  训练快，胜率 52%（刚好够用）
# 128: 训练快，胜率 51%（学得更快但容易"死记硬背"）
```

选择标准：
- **训练速度**：维度越大，每次计算越慢
- **模型大小**：维度越大，保存的权重越多
- **最终性能**：胜率、收敛速度、是否过拟合

井字棋用 64 是一个平衡点——够快、够小、够准。

### 过拟合：学得太多也是问题

隐藏层太大会有副作用：**过拟合**（overfitting）。

过拟合的意思是网络"死记硬背"了训练数据，而不是"理解"了规律。举个例子：

```
训练时见过的局面：
 X | O |
-----------
   | X |
-----------
   |   | O
→ 网络记住了"这个局面下位置 2 最好"

训练时没见过的局面：
 O | X |
-----------
   | O |
-----------
 X |   |
→ 网络懵了，因为它没见过这个局面
```

如果隐藏层太大（比如 1024），网络有足够的容量把所有训练局面都"背下来"，但遇到新局面就不会了。

如果隐藏层适中（比如 64），网络被迫"理解"规律（比如"形成双威胁必胜"、"中心位置最重要"），而不是死记硬背。这样遇到新局面也能应对。

这就是为什么我们说 64 是"平衡点"——它足够大，能学到关键模式；又不够大，不会死记硬背。

### 小结

| 考量 | 64 的优势 |
|------|----------|
| 计算便利 | 2^6，内存对齐、矩阵运算高效 |
| 容量足够 | 比输入（18）和输出（9）都大，有足够思考空间 |
| 任务匹配 | 井字棋简单，64 够用，不需要 256/512 |
| 行业惯例 | ResNet 之后的主流选择，经过大量实践验证 |
| 避免过拟合 | 适中大小，被迫"理解"而非"死记" |

下次你看到别人的网络用了 128 或 256，就知道这不是随便选的——它背后有计算效率、任务复杂度、行业惯例等多重考量。而对于你自己的项目，最好的办法还是实验对比，找到最适合你任务的那个"魔法数字"。
