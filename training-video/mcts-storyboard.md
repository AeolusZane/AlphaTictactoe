# MCTS 可视化视频 - 场景分镜设计

## 视频概述
展示蒙特卡洛树搜索如何通过模拟对局来评估走法，核心是"选择-扩展-回溯"循环。

## 场景设计（总时长约 90 秒）

### 场景 1：标题页（0-5秒）
- 标题："蒙特卡洛树搜索"
- 副标题："通过模拟对局找到最佳走法"
- 动画：淡入

### 场景 2：问题引入（5-15秒）
- 左侧：当前棋盘局面（黑棋要走）
- 右侧：3个合法走法（位置 2, 4, 6）
- 文字："黑棋该走哪里？如何评估每个走法的好坏？"
- 配音：解释问题

### 场景 3：核心思想（15-25秒）
- 动画：从当前局面出发，画出多条模拟对局路径
- 文字："让 AI 自己跟自己下很多局，统计每个走法的胜率"
- 配音：解释蒙特卡洛方法的核心思想

### 场景 4：搜索树结构（25-35秒）
- 展示搜索树：根节点（当前局面）→ 子节点（各种走法）→ 孙节点...
- 每个节点显示：访问次数 / 累计价值
- 文字："搜索树：每个节点代表一个局面"
- 配音：解释树结构

### 场景 5：选择阶段（35-50秒）
- 动画：从根节点开始，沿 UCB 最高的路径高亮走到叶子
- 公式显示：UCB = value + c * prior * sqrt(parent_visits) / (1 + visits)
- 文字："选择：沿 UCB 最高的路径走到叶子节点"
- 配音：解释 UCB 公式平衡探索和利用

### 场景 6：扩展阶段（50-65秒）
- 动画：叶子节点扩展出子节点，神经网络评估给出价值
- 显示：神经网络输出 policy（先验概率）和 value（胜率估计）
- 文字："扩展：用神经网络评估叶子节点，创建子节点"
- 配音：解释神经网络的作用

### 场景 7：回溯阶段（65-80秒）
- 动画：从叶子往根节点回溯，更新路径上所有节点的 visits 和 value_sum
- 注意：每上一层翻转符号（对手视角）
- 文字："回溯：把评估值传回路径上所有节点"
- 配音：解释回溯和符号翻转

### 场景 8：多次模拟（80-100秒）
- 动画：快速播放多次模拟，树逐渐生长
- 显示：模拟次数计数器（1, 10, 50, 100）
- 根节点的子节点访问次数逐渐分化
- 文字："重复 100 次模拟，统计每个走法的访问次数"
- 配音：解释多次模拟的意义

### 场景 9：最终选择（100-110秒）
- 动画：高亮访问次数最多的走法
- 显示：visit_counts = {2: 15, 4: 70, 6: 15} → 选择位置 4
- 文字："选择访问次数最多的走法"
- 配音：解释为什么用访问次数而不是价值

### 场景 10：总结（110-120秒）
- 流程图：选择 → 扩展 → 回溯 → 重复 → 选择最佳
- 文字："MCTS 通过模拟对局，找到最有潜力的走法"
- 配音：总结

## 视觉设计

### 搜索树可视化
- 节点：圆形，显示 visits / value
- 边：线条粗细表示 prior 概率
- 颜色：
  - 当前路径高亮：黄色
  - 已访问节点：绿色（正价值）/ 红色（负价值）
  - 未访问节点：灰色

### 棋盘
- 3x3 网格，黑棋绿色，白棋红色
- 当前要走的位置闪烁

### 公式
- UCB 公式用 LaTeX 风格渲染
- 关键项高亮

## 配音脚本

### 场景 2
"黑棋该走哪里？位置 2、4、6 都是合法走法。如何评估每个走法的好坏？"

### 场景 3
"蒙特卡洛方法的核心思想：让 AI 自己跟自己下很多局，统计每个走法的胜率。胜率高的走法，就是好走法。"

### 场景 4
"搜索树的每个节点代表一个局面。根节点是当前局面，子节点是走一步后的局面。每个节点记录访问次数和累计价值。"

### 场景 5
"选择阶段：从根节点开始，沿 UCB 最高的路径走到叶子节点。UCB 公式平衡探索和利用：既要选胜率高的走法，也要探索访问少的走法。"

### 场景 6
"扩展阶段：到达叶子节点后，用神经网络评估这个局面的胜率，并创建子节点。神经网络给出先验概率，指导后续搜索。"

### 场景 7
"回溯阶段：把叶子节点的价值传回路径上所有节点。注意每上一层要翻转符号，因为对手的好事是我们的坏事。"

### 场景 8
"重复这个过程 100 次。随着模拟次数增加，好的走法会被更多次访问，差的走法逐渐被淘汰。"

### 场景 9
"最终，选择访问次数最多的走法。这里位置 4 被访问了 70 次，远多于其他走法，所以选择位置 4。"

### 场景 10
"蒙特卡洛树搜索通过模拟对局，找到最有潜力的走法。这就是 AlphaGo 的核心算法之一。"
